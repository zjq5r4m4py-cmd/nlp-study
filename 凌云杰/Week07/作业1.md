# 作业1

## 1. BERT文本分类与实体识别的关系 + 各自Loss
### 1.1 任务核心关系
BERT 是基于Transformer架构的预训练语言模型，**文本分类**和**实体识别（NER）** 是其两大核心下游任务，二者的核心关联与差异如下：
- 底层共享：两者均复用BERT的上下文语义编码层，利用BERT输出的token级向量表征完成任务；在联合训练（Joint Training）中可共享BERT主干参数，实现任务间信息互补（如文本分类的意图信息辅助NER识别领域专属实体，NER的实体信息辅助文本分类更精准）；
- 粒度差异：
  - 文本分类：**序列级任务**，目标是给整段文本分配1个/多个类别标签（如意图分类、情感分类），聚焦“文本整体语义”；
  - 实体识别（NER）：**Token级任务**，目标是给文本中每个字/词分配实体标签（如B-PER、I-ORG、O等），聚焦“文本局部实体语义”。

### 1.2 各自Loss选择
| 任务类型       | 核心Loss类型                | 补充说明                                                                 |
|----------------|-----------------------------|--------------------------------------------------------------------------|
| 文本分类       | 交叉熵损失（CrossEntropyLoss） | 基于BERT的`[CLS]` token输出向量做多分类，计算真实类别与预测类别的交叉熵；若为多标签分类，可替换为BCEWithLogitsLoss。 |
| 实体识别（NER）| ① 基础版：CrossEntropyLoss<br>② 优化版：CRF Loss | ① 逐Token做分类，直接计算每个Token标签的交叉熵（未考虑标签间约束，如B-PER后不能直接接O）；<br>② CRF Loss 引入标签转移概率，优化序列标注的全局合理性，是NER工业界主流Loss。 |

## 2. 多任务Loss=seq_loss+token_loss的坏处 + 训练不平衡处理
### 2.1 直接求和的核心坏处
多任务训练中直接将文本分类Loss（seq_loss）和实体识别Loss（token_loss）相加，存在以下关键问题：
1. **Loss量级失衡**：token_loss因累加所有Token的损失，数值通常远大于seq_loss（序列级单值），导致优化过程被“大Loss任务”主导，另一任务的梯度被淹没，训练效果差；
2. **收敛速度不匹配**：两个任务的难度、收敛速率不同（如文本分类收敛快，NER收敛慢），简单任务先收敛后梯度趋近于0，会拖慢/抑制难任务的优化；反之难任务的大Loss会导致简单任务过拟合；
3. **无自适应调整**：固定权重（均为1）无法根据训练阶段动态调整任务优先级（如初期兼顾两个任务，后期侧重效果差的任务）。

### 2.2 训练不平衡的处理方法
| 方法类型               | 具体实现方式                                                                 |
|------------------------|------------------------------------------------------------------------------|
| 加权损失（核心方案）| 给不同任务Loss添加权重系数：`loss = α*seq_loss + β*token_loss`；α/β可手动调优，或根据验证集效果、Loss方差自适应调整； |
| Loss归一化             | 对每个任务Loss做量级对齐：<br>- 基础版：token_loss ÷ 文本Token总数、seq_loss ÷ 样本数；<br>- 进阶版：Z-Score标准化（(Loss-均值)/标准差）； |
| 梯度层面优化           | 对每个任务的梯度单独裁剪（Gradient Clipping），或对总梯度做L2归一化，避免单一任务梯度主导参数更新； |
| 阶段式训练             | 先单独预训练每个任务（让参数适配各自任务），再联合训练；或初期侧重难任务（如NER），后期平衡权重； |
| 自适应权重             | 动态调整α/β：如根据任务Loss的下降速率（Loss下降慢的任务增加权重）、温度系数、强化学习优化权重； |
| 模型结构优化           | 采用MMoE（混合专家模型），给不同任务分配专属“专家网络”，缓解任务间的优化冲突； |
